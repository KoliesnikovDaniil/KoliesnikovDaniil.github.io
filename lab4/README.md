# Лабораторна робота №4

## Тема
Автентифікація та підключення до бази даних.

## Варіант 9
Веб-сайт платформи для організації волонтерських ініціатив.

## Мета роботи
Навчитись реалізовувати автентифікацію користувачів, а також підключення до бази даних для зберігання, читання та управління даними вебзастосунку.

## Реалізовано
У лабораторній роботі створено вебзастосунок VolunteerHub, який демонструє роботу з автентифікацією користувачів та базою даних.

Реалізовані можливості:

1. Створено форми реєстрації та входу користувача через email і пароль.
2. Реалізовано механізм виходу користувача із системи.
3. Додано відстеження стану автентифікації користувача.
4. Для неавтентифікованого користувача заблоковано можливість залишати оцінки.
5. Для автентифікованого користувача реалізовано можливість залишати оцінку ініціативи від 1 до 5 зірок.
6. Створено демонстраційний список волонтерських ініціатив, який імітує читання даних з Firebase Firestore.
7. Реалізовано збереження оцінок, яке імітує запис у базу даних.
8. Додано пояснення щодо підключення реального Firebase через firebaseConfig.

## Порядок виконання роботи

### Крок 1. Налаштування автентифікації користувача
На першому етапі було розглянуто налаштування автентифікації користувачів за допомогою Firebase Authentication. Для цього необхідно створити проєкт у Firebase Console, додати вебзастосунок і увімкнути спосіб входу через email та пароль.

У реалізованому проєкті створено окремий блок для автентифікації, який містить форму реєстрації, форму входу та кнопку виходу. Також додано блок відображення поточного статусу користувача.

### Крок 2. Підключення автентифікації до клієнтської частини
На другому етапі було підготовлено підключення Firebase SDK до клієнтської частини. У файлі script.js додано приклад firebaseConfig, який потрібно замінити на власні дані з Firebase Console.

У проєкті реалізовано функції реєстрації, входу та виходу. У демонстраційному режимі ці дії імітуються за допомогою localStorage, щоб сайт можна було запустити без реального Firebase-проєкту. Після додавання справжнього firebaseConfig можна використати методи createUserWithEmailAndPassword, signInWithEmailAndPassword, signOut та onAuthStateChanged.

### Крок 3. Додавання функції реєстрації
На третьому етапі було створено форму реєстрації користувача. Вона містить поля для введення email і пароля.

Після натискання кнопки «Зареєструватися» JavaScript перевіряє, чи заповнені поля і чи має пароль достатню довжину. Якщо дані коректні, користувач створюється та автоматично входить у систему. Після цього на сторінці оновлюється статус автентифікації.

### Крок 4. Додавання функції входу в систему та виходу
На четвертому етапі було реалізовано форму входу користувача. Вона також містить поля для email і пароля.

Після введення даних система перевіряє, чи існує такий користувач. Якщо облікові дані правильні, користувач входить у систему. Також реалізовано кнопку «Вийти», яка завершує поточну сесію та змінює статус користувача на «Користувач не увійшов».

### Крок 5. Підключення до бази даних
На п’ятому етапі було підготовлено підключення до Firebase Firestore. Для реального використання потрібно у Firebase Console створити Firestore Database, вибрати режим доступу та створити колекцію initiatives.

У виконаному проєкті список ініціатив розміщено у JavaScript-масиві, який імітує отримання даних з бази. Кожна ініціатива має назву, дату, місце проведення, тип активності, опис, кількість потрібних волонтерів та рейтинг.

### Крок 6. Читання та запис даних
На шостому етапі було реалізовано читання та запис даних. Читання демонструється через відображення списку волонтерських ініціатив на сторінці. Запис демонструється через форму оцінювання ініціатив.

Оцінки можуть залишати тільки автентифіковані користувачі. Якщо користувач не увійшов у систему, при спробі залишити оцінку виводиться повідомлення про необхідність входу. Якщо користувач автентифікований, його оцінка додається до списку збережених оцінок.

## Контрольні запитання

### 1. Що таке JWT і як він використовується для автентифікації користувачів у веб-додатках?
JWT, або JSON Web Token, — це токен, який використовується для безпечної передачі інформації між клієнтом і сервером. Після успішного входу сервер створює токен і передає його клієнту. Далі клієнт надсилає цей токен разом із запитами, щоб підтвердити свою особу. JWT часто використовується для автентифікації без збереження сесії на сервері.

### 2. Які основні відмінності між JWT та OAuth?
JWT — це формат токена, який містить інформацію про користувача та може використовуватися для підтвердження автентичності запитів. OAuth — це протокол авторизації, який дозволяє користувачу входити через сторонні сервіси, наприклад Google або Facebook, не передаючи пароль самому вебзастосунку. JWT частіше використовується для передачі даних автентифікації, а OAuth — для делегованого доступу.

### 3. Які сервіси можна використовувати для розгортання клієнтської та серверної частини веб-додатка?
Для клієнтської частини можна використовувати GitHub Pages, Firebase Hosting, Netlify або Vercel. Для серверної частини можна використовувати Render, Railway, Heroku, Firebase Functions або Google Cloud. GitHub Pages добре підходить для статичних сайтів, Firebase Hosting — для вебзастосунків з інтеграцією Firebase, а Vercel і Netlify зручні для React, Vue та інших frontend-проєктів.

### 4. Як працює процес автентифікації користувачів за допомогою Firebase?
Firebase Authentication надає готові методи для реєстрації, входу, виходу та відстеження стану користувача. Спочатку у Firebase Console потрібно створити проєкт і увімкнути метод входу, наприклад email/password. Потім у коді підключається Firebase SDK, налаштовується firebaseConfig і використовуються функції createUserWithEmailAndPassword, signInWithEmailAndPassword, signOut та onAuthStateChanged.

### 5. Які основні кроки необхідні для безпечного зберігання токенів автентифікації у веб-додатку?
Токени потрібно зберігати так, щоб зменшити ризик їх викрадення. Варто використовувати HTTPS, обмежувати час життя токенів, застосовувати HttpOnly cookies для важливих токенів, не зберігати конфіденційні дані у відкритому вигляді в localStorage, регулярно оновлювати токени та налаштовувати правила доступу до бази даних.

## Висновок
У результаті виконання лабораторної роботи було створено вебзастосунок з формами реєстрації та входу, перевіркою стану користувача, можливістю виходу із системи, відображенням даних ініціатив і збереженням оцінок. Робота демонструє основні принципи автентифікації користувачів і взаємодії клієнтської частини з базою даних Firebase Firestore.
