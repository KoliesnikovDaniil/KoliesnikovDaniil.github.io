# Лабораторна робота №3

## Тема
Фронтенд-фреймворки.

## Варіант
Варіант 9 — веб-сайт платформи для організації волонтерських ініціатив.

## Мета роботи
Отримати практичний досвід роботи з сучасним фронтенд-фреймворком, навчитися створювати веб-сторінки за допомогою компонентів, керувати станом компонентів та реалізовувати маршрутизацію між сторінками без перезавантаження браузера.

## Обраний фреймворк
Для виконання лабораторної роботи використано React. Додаток реалізовано як односторінковий веб-застосунок, у якому інтерфейс поділено на окремі компоненти, а перемикання між сторінками виконується без повного перезавантаження сторінки.

## Реалізована функціональність

У роботі створено сайт VolunteerHub — платформу для організації волонтерських ініціатив. Додаток містить три основні сторінки:

1. **Доступні ініціативи** — сторінка зі списком волонтерських проєктів.
2. **Мої ініціативи** — сторінка зі списком ініціатив, до яких приєднався користувач.
3. **Про нас** — інформаційна сторінка про організацію та її місію.

Для варіанту 9 реалізовано такі вимоги:

- створено компоненти для картки ініціативи, форми реєстрації та списку приєднаних ініціатив;
- у компоненті «Доступні ініціативи» додано фільтри та сортування за датою, місцем і типом активності;
- реалізовано маршрутизацію між сторінками «Доступні ініціативи», «Мої ініціативи» та «Про нас» без перезавантаження сторінки;
- використано стан React для збереження списку ініціатив, обраної ініціативи та списку проєктів, до яких користувач приєднався;
- реалізовано кнопку «Приєднатися», яка змінює стан кнопки та додає ініціативу до списку користувача;
- реалізовано форму реєстрації з перевіркою введених даних.

## Компоненти додатку

### Header
Компонент верхньої частини сайту. Містить назву платформи, короткий опис і навігаційні кнопки для переходу між сторінками.

### FilterPanel
Компонент фільтрів. Дозволяє користувачу сортувати ініціативи за датою, місцем і типом активності.

### InitiativeCard
Компонент картки волонтерської ініціативи. Відображає назву, опис, дату, місто, тип активності, кількість потрібних волонтерів і кнопки для приєднання або відкриття форми.

### RegistrationForm
Компонент форми реєстрації на ініціативу. Містить поля для імені, електронної пошти та коментаря. Після перевірки даних дозволяє приєднатися до обраної ініціативи.

### AvailableInitiatives
Компонент сторінки доступних ініціатив. Відповідає за відображення списку проєктів, застосування фільтрів і сортування.

### JoinedInitiativesList
Компонент сторінки «Мої ініціативи». Показує список проєктів, до яких користувач уже приєднався.

### About
Компонент інформаційної сторінки про організацію.

### App
Головний компонент додатку. Зберігає основний стан програми, керує маршрутизацією та передає дані дочірнім компонентам.

## Керування станом

У React-додатку використано хук `useState`. Він застосовується для збереження:

- поточного маршруту;
- списку ініціатив;
- списку ідентифікаторів ініціатив, до яких приєднався користувач;
- обраної ініціативи для форми реєстрації;
- даних форми.

При натисканні кнопки «Приєднатися» оновлюється стан `joinedIds`, а також зменшується кількість потрібних волонтерів у відповідній ініціативі.

## Маршрутизація

Маршрутизацію реалізовано у форматі SPA за допомогою hash-навігації. Користувач може переходити між сторінками:

- `#/initiatives`
- `#/my`
- `#/about`

Під час переходів сторінка браузера не перезавантажується, змінюється тільки вміст основної області додатку.

## Контрольні запитання

### 1. Що таке компонентно-орієнтована архітектура у фронтенд-фреймворках, і які її переваги?
Компонентно-орієнтована архітектура — це підхід, за якого інтерфейс додатку поділяється на невеликі незалежні частини — компоненти. Кожен компонент має власну структуру, стилі та логіку. Переваги такого підходу: повторне використання коду, зручна підтримка, простіше тестування та легше масштабування додатку.

### 2. Які ключові відмінності між Angular, React та Vue у способі керування станом компонента?
Angular керує станом через властивості класу компонента, які автоматично відображаються у шаблоні. React використовує хуки, наприклад `useState`, для створення та оновлення стану. Vue використовує функцію `data()` або Composition API, а зміни стану автоматично відображаються в шаблоні.

### 3. Що таке Virtual DOM і як він покращує продуктивність у React і Vue?
Virtual DOM — це легка копія реального DOM у пам’яті. Коли стан додатку змінюється, фреймворк спочатку оновлює Virtual DOM, порівнює зміни з попереднім станом і вносить у реальний DOM тільки необхідні оновлення. Це зменшує кількість дорогих операцій з DOM і покращує продуктивність.

### 4. Як працює маршрутизація у фронтенд-фреймворках, і які бібліотеки використовуються для неї у Angular, React та Vue?
Маршрутизація дозволяє перемикатися між різними сторінками або представленнями додатку без повного перезавантаження сторінки. В Angular для цього використовується RouterModule, у React — React Router, у Vue — Vue Router.

### 5. Що таке директиви у Angular та Vue, і як вони відрізняються від умовного рендерингу у React?
Директиви — це спеціальні атрибути, які додають поведінку HTML-елементам. В Angular використовуються `ngIf`, `ngFor`, `ngClass`, у Vue — `v-if`, `v-for`, `v-model`. У React окремих директив немає: умовний рендеринг реалізується через JavaScript-вирази всередині JSX.
